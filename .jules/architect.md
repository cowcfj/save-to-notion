# Architect's Journal

## 2026-07-03 - [上帝模組與混合關注點模式]

**Pattern:** 專案中存在數個「上帝模組」(God Modules)，這些模組將 UI 狀態協調、直接的 `chrome.storage` 存取、以及特定領域的業務邏輯全部混合在單一檔案內。另一個常見的模式是將原生的 DOM 解析與複雜的驗證規則和批次處理管線交織在一起。
**Why it exists:** 這通常是由於功能隨時間自然增長所致，例如最初簡單的功能（獲取頁面圖片或儲存設定）隨著邊界情況、容錯處理和效能優化的加入而膨脹，且缺乏相對應的結構重構。
**Refactor verdict:** Approved. 建議針對具備完善測試覆蓋率的模組進行解耦。涉及敏感流程（如 OAuth）的檔案已被標記為 Deferred，因為當前的安全規範禁止觸碰這類路徑。
